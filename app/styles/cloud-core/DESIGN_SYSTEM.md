# Cloud Core - Design System

## Visual Ethos: Global Infrastructure
The "Global Infrastructure" theme is focused on the massive scale, high speed, and interconnected nature of modern cloud computing. It emphasizes the "core" of the internet—massive data centers, fiber optic networks, and the relentless flow of information. The aesthetic is clean, technical, and high-tech, using deep blues, glowing cyans, and silver/white accents to represent the purity and efficiency of data.

- **Scale & Magnitude:** UI elements feel part of a larger, global system. Layouts are expansive and interconnected.
- **Data Flow:** Information is represented as glowing streams and "motes" of data moving at high velocity.
- **Interconnectedness:** SVG path lines connect various UI nodes, simulating the physical connections of a global network.
- **Clarity & Efficiency:** High contrast, technical typography, and a "clean room" aesthetic.

## Design Tokens

### Colors
- **Background:** `#020617` (Deep Midnight) - The vastness of the digital space.
- **Surface:** `#0f172a` (Infrastructure Slate) - Heavy, solid server-like containers.
- **Accent 1:** `#38bdf8` (Data Cyan) - Primary flow lines and active nodes.
- **Accent 2:** `#818cf8` (Packet Indigo) - Secondary data packets and secondary actions.
- **Glow:** `rgba(56, 189, 248, 0.4)` (Cyan Pulse) - Interactive and high-velocity highlights.
- **Text (High):** `#f8fafc` (Fiber White)
- **Text (Low):** `#64748b` (Ether Grey)

### Typography
- **Primary:** `Inter` - For technical precision and readability.
- **Display:** `JetBrains Mono` - For code-like clarity and a technical infrastructure feel.

### Spacing & Layout
- Expansive, grid-aligned layouts with thin, glowing lines separating sections.
- Use of "data nodes" (circles/icons) at connection points.

## Motion Language (Kinetic Design Protocol)

### 1. Velocity-Responsive Data Motes
- Background particles ("motes") that increase in speed, density, and length based on scroll velocity.
- Creates a "warp speed" effect when scrolling rapidly through the "cloud core".

### 2. SVG Path Connectivity
- Dynamic SVG paths that connect specific UI nodes (e.g., icons, headings).
- Paths animate and "pulse" with Data Cyan highlights as the user scrolls.
- The connections represent the routing of data through the infrastructure.

### 3. Kinetic Infrastructure Spring
- UI elements use a "precise infrastructure" spring.
- **Token:** `stiffness: 400, damping: 25, mass: 1`
- Feels solid and well-engineered, with minimal but precise recoil.

### 4. Interconnected Transitions
- When nodes enter the viewport, the connecting paths "draw" themselves in sync with the node's arrival.
